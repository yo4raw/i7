# 0006: イベント SNS 共有パネルを高密度レイアウトへ

- 日付: 2026-06-11
- ステータス: 承認（2026-06-11 ユーザー承認）

## 背景

イベント詳細配下の SNS 共有ページ（`events/[id]/share/ur|ssr`）は 3〜5 列・`gap-2` のグリッドにカード名 2 行 + キャラ名のテキストを付けて表示しており、SNS 共有用の画像としては疎すぎる。非公式 DB の特効まとめ画像のような高密度な並びにしたい、という要望があった。要件は「スキル・衣装名・対象イベント名が分かりやすいこと」「判定縮小が目立つこと」。

## 決定

ブラウザ上のモックアップ比較（視覚コンパニオン）でユーザーが選択した構成を採用する:

- **案 A「参考画像準拠」**: 6 列固定・`gap-px` のグリッドで、各セルは「衣装名（上ラベル）/ サムネイル + 属性バッジ / スキル種別セル / 発動条件セル」の縦積み
- 発動条件セルは種別を問わず**黄色で統一**（色分けしない）
- **判定縮小はスキル種別セルをピンク背景 + 白文字**で強調
- 属性バッジは画像左上に維持。レアリティバッジ・カード名 2 行・キャラ名行は削除（ページが UR/SSR 別のため冗長）
- 列数はブレークポイントで変えず常に 6 列（ダウンロード画像の見た目を端末によらず一定にする）
- ヘッダー帯（イベント名・期間）と特効セクション見出しは現行維持

## 検討した代替案

- 案 B: オーバーレイ型（画像下部に半透明バーで最高密度）— 衣装イラストが隠れるため不採用
- 案 C: 1 行コンパクト（「スコアUP/コンボ」形式）— 視認性で案 A に劣るため不採用
- 列数 8 / 10 列 — 参考画像と同等の密度だが文字が拡大前提のサイズになるため不採用
- 発動条件の色分け（コンボ=黄 / タイマー=水色等）— ユーザー判断で黄色統一とした
