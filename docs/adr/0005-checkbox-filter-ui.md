# 0005: 衣装一覧フィルタを select multiple からチップ+折りたたみへ

- 日付: 2026-06-11
- ステータス: 承認（2026-06-11 ユーザー承認）

## 背景

衣装一覧の絞り込み条件（レアリティ・属性・キャラクター・スキルタイプ・特効）は `<select multiple>` で実装されている。複数選択に Ctrl/Cmd+クリックの知識が必要、モバイルでは OS 依存の UI になる、選択状態が一覧しにくい、という操作性の問題があった。

## 決定

モックアップ 3 案を比較し、**ハイブリッド方式（案 C）** を採用する:

- 選択肢の少ないレアリティ・属性・特効はトグルチップを常時表示（1 タップで切替、状態が常に見える）
- 選択肢の多いキャラクター・スキルタイプはネイティブ `<details>` 折りたたみ内のチップ（選択件数バッジ付き、選択済み条件があるときのみ初期展開）
- キャラクターはグループ（IDOLiSH7 / TRIGGER / Re:vale / ŹOOĻ）単位の一括選択チップを併設
- フィルタ状態・絞り込みロジック・URL パラメータ同期は変更しない（UI 置換のみ)

## 検討した代替案

- 案 A: 全条件チップ常時表示 — 操作は最速だがスキルタイプ 11 種の長文チップで縦に長くなり、一覧が下に押し出されるため不採用
- 案 B: ドロップダウン式チェックボックス + 選択中チップ — 現状と同等の省スペースだが、切替に 2 タップかかり、ポップオーバーの開閉制御（外側クリック検知）で実装が複雑になるため不採用
