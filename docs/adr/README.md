# Architecture Decision Records (ADR)

設計・仕様に関する意思決定の記録。決定の背景と理由を後から追えるようにする。

## 運用ルール

- ファイル名: `NNNN-<kebab-case-title>.md`（連番 4 桁）
- ステータス: `提案` → `承認` → 実装後も記録は残す。覆った場合は `却下` / `破棄` に更新し、理由を追記する
- 1 ADR = 1 つの意思決定。実装の詳細ではなく「何を・なぜ決めたか」を書く
