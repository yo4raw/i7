# 0002: 編成組合計算における固定ブローチの扱い

- 日付: 2026-06-11
- ステータス: 承認（現状動作の確認・変更なし）

## 背景

「編成組合計算（max-score-finder）で固定ブローチが考慮されていないように見える。オート専用は対象外とし、三属性編成での属性値付与や、特定キャラ + 特定属性での属性値付与は計算対象にしたい」という指摘があった。

## 調査結果

要望の挙動は**すべて実装済み**であることを実データ（テストフィクスチャ）で確認した。

- 固定ブローチの条件判定は `src/lib/score/broachResolver.ts` に集約され、finder の評価経路（`evaluateDeck` → `computeTeam`）で適用される
- 種類 8（オート専用）は常に無効（計算対象外）。結果画面でも取り消し線 + 「計算対象外」表示
- 種類 7（全属性編成で属性値付与、上限あり）: finder 経路でスコア反映を実測確認（+600×3 属性 → スコア +17,665）
- 種類 5（特定キャラ + 特定属性で属性値付与、対象枚数でスケール）: 同じく実測確認（+6,497）
- 種類 1 / 4 / 6 / 9（無条件属性 UP / グループ統一 / 上限あり属性 UP / 楽曲指定スコア UP）も適用される
- 探索結果の衣装詳細テーブルにブローチ加算値が紫の `+値` として表示される

## 決定

実装変更は行わない。

## 補足（誤解の原因）

`maxScoreFinder.ts` とテストにある「探索条件は…**共有ブローチなしで固定**」というコメントは、全カード共通で付け替え可能な**共有ブローチ**を探索対象外にしている意味であり、カード固有の**固定ブローチ**は除外されていない。紛らわしい用語が誤解の原因と推定される。
