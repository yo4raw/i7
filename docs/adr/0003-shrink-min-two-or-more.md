# 0003: 判定縮小条件を「ちょうど2枚」から「2枚以上」へ

- 日付: 2026-06-11
- ステータス: 承認（2026-06-11 ユーザー承認）

## 背景

編成組合計算の追加条件「判定縮小2枚編成」（`shrinkPairOnly`）は、現在デッキ 6 枠中の縮小スキル持ちを**ちょうど 2 枚**に固定して探索する。

しかし:

- 縮小 1 枚ではカバー率が確実に 100% にならない（2 枚が実用上の最低ライン）
- 2 枚で 100% を超えても、縮小衣装の属性値がイベント特効で高いパターンでは **3 枚以上**入れた方がスコアが伸びる可能性がある

## 決定（設計）

「ちょうど 2 枚」を「**2 枚以上**（上限なし）」に変更する。

| モード | 変更後の規則 |
| ------ | ------------ |
| 通常（特効候補のみ） | 6 枠合計 2 枚以上のみ列挙（6 枚全縮小も可） |
| 所持衣装縛り | own5（センター+メンバー）≥ 2 → フレンドは全候補 / own5 ≤ 1 → 縮小フレンドのみ（除外はせず、最良の縮小フレンドが推薦される形を維持） |
| フレンド差し替え Top5 (`evaluateFriendSwap`) | 固定 5 枠の縮小 ≥ 2 → 全候補プール / ≤ 1 → 縮小プール（探索時と同一規則） |

実装方針: チャンク構造（s2 = センター/フレンドペア内の縮小枚数）は不変のまま、メンバー 4 枠の縮小枚数 k を `max(0, 2 − s2)`〜4 のループに一般化する。定数 `SHRINK_PAIR_TARGET` は意味が「固定枚数」から「最低枚数」に変わるため `SHRINK_MIN` にリネームする。

## 影響

- 探索空間が広がる（組合せ数は探索開始前に UI 表示されるため利用者が事前に判断可能）
- UI 文言（チェックボックスラベル・説明・0 件時メッセージ）と単体テストの期待値を新仕様に更新する

## 検討した代替案

- 最低枚数を UI パラメータ化（1〜6 選択式）: 要望は「2 以上」固定で足りるため YAGNI で見送り
- 条件なしで全探索し結果側でフィルタ: 縮小 0〜1 枚の組合せまで評価する計算の無駄が大きく却下
