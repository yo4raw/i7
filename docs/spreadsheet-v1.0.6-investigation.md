## 3. ★重要: 「最大値を期待値として記載している可能性」

スコア計算シートの「スキルオプション」に以下のトグルがある:

| トグル | 値 | 意味（暫定解釈・要検証） |
| --- | --- | --- |
| スコアアップフル発動 / スコアアップ確率UP | TRUE/FALSE | フル発動＝**最大値**、確率UP＝**期待値** |
| 縮小フル発動 / 20ノーツ加算なし(β版) | TRUE/FALSE | フル発動＝縮小**全発動（最大）** |

→ スプレッドシートの「最終リザルト」は**トグル次第で最大値にも期待値にもなる**単一値。
ユーザー指摘の「スプレッドシートは最大値を期待値として記載している可能性」とは、
**フル発動トグル ON のまま出た値を期待値と取り違えるリスク**を指す。
差分検証時は「スプレッドシートのどのトグル状態の値か」を必ず明示して突き合わせること。

## 4. 現行 engine.ts の出力区分（対応関係の候補）

| engine.ts 関数 | 役割 | スプレッドシート側の対応候補 |
| --- | --- | --- |
| `calcExpectedScore` | 算術期待値（決定論） | スキルオプション＝確率UP の最終リザルト |
| `calcMaxScore` | 最大値（縮小・スコアアップ全発動） | スキルオプション＝フル発動の最終リザルト |
| `calcMinScore` | 最低値（縮小全不発） | （スプレッドシートに明確な対応なし。要確認） |
| `runSimulation` | モンテカルロ分布 | スプレッドシートに対応なし（engine 独自） |

## 5. 検証方針（ユーザー合意済み）

- **当面はスプレッドシート（v1.0.6）を参照正**として差分を抽出し、差分が出たら個別に「engine とスプレッドシートのどちらが実機に近いか」を判断する（方針 1 + 4 の折衷）。
- **期待値・最大値**はスプレッドシートに一致させることを目標とする。
- **分布**は engine.ts の乱数モンテカルロが出す独自結果とし、スプレッドシートに合わせる対象外。

## 5-2. たちばなさんロジックの削除（ユーザー指示）

「たちばなさんロジック」（`tachibana/`）はスプレッドシートと同じトグル
（`scoreUpFullActivation` / `shrinkFullActivation` / `excludeNotes20` / `specialEffectActive`）を持つ
**別実装の代替計算**で、本タスクの「本体エンジンを v1.0.6 に合わせる」方針と役割が重複する。
ユーザー指示により削除する。削除範囲は以下の 2 箇所に完全に閉じている:

| 対象 | 内容 |
| --- | --- |
| `src/lib/score/tachibana/` | ディレクトリごと削除（`constants.ts` / `engine.ts` / `index.ts` / `types.ts`） |
| `src/components/ScoreCalc.svelte` | `computeTachibanaResult` import・`renderTachibanaResult()`・UI マークアップ（チェックボックス群 L1611-1640・結果セクション L1828）・state 保存/復元（L1147-1169 / L1221-1233）・イベントリスナー（L1491 付近） |

`i7_score_calc_state` に残る旧 `tachibana` キーは復元側を削除すれば単に無視されるため、マイグレーション不要。
