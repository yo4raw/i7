スキル (BAD→Perfect) はスコア寄与なし、固定ブローチも属性値加算型のため `broachScoreBonus = 0`。
したがって仕様通りに計算した場合 `calcMinScore = calcMaxScore = MC 全 scores = 163,242` となる。

##　最終スコア

基本スコア **163,242** に対し、以下のスコアアップ要素が存在する場合は加算される:

- SCOREUPアシストが `true` なら `163,242 * 0.12` を加算
  - 加算値: `floor(163,242 × 0.12)` = 19,589
  - 適用後スコア: 163,242 + 19,589 = **182,831**
- SCOREUPバッジ倍率に 15 が入力されていたら `163,242 * 0.15` を加算
  - 加算値: `floor(163,242 × 0.15)` = 24,486
  - 適用後スコア: 163,242 + 24,486 = **187,728**

### 両方同時に適用された場合

加算率を合算 (12% + 15% = 27%) して基本スコアに加算:

- 加算値: `floor(163,242 × 0.27)` = 44,075
- 最終スコア: 163,242 + 44,075 = **207,317**

### 計算例まとめ

| 適用条件 | 加算率 | 加算値 | 最終スコア |
|---------|------:|------:|----------:|
| なし (基本スコアのみ) | 0% | 0 | 163,242 |
| SCOREUPアシストのみ | 12% | 19,589 | 182,831 |
| SCOREUPバッジ (15) のみ | 15% | 24,486 | 187,728 |
| アシスト + バッジ (15) | 27% | 44,075 | **207,317** |


# SongsのID2のMonsterGenerationでデッキがセンターにCardのID3414でフレンドにID1952の2枚構成だった場合のスコア計算方法

- センターの Card ID3414 は「BAD以上をPerfectに変更」なのでスコア寄与なし
- フレンドの Card ID1952 は「判定縮小（Perfect）」スキル付きのため **スコアに影響する**
  - SL5: 20ノーツごとに 40% 発動、4 秒持続、対象ノートを 1.6 倍
- 縮小スキルの確率発動により MC シミュレーションのスコア分布が 1 点に収束せず、min / max / 期待値 / MC に差が出る

| ID1952 (記念日2024 四葉環) | フレンド | Melody | UR | 4,691 | 4,243 | 7,666 | 判定縮小 Perfect (SL5: 20ノーツ毎 40% / 4秒 / 1.6倍) |

## 縮小スキル発動時の最終スコア

Card 1952 の「判定縮小（Perfect）」が発動している間、notes_20 以外のノートのスコアが **×1.6 倍** になる (`SHRINK_MULTIPLIER = 1.6`)。

実装 (`calcNoteScore` + `calcShrinkCoverage`) での各算出値:

| ケース | スコア | 差分 (対 基本スコア) |
|--------|------:|------:|
| 基本スコア (縮小全不発 = `calcMinScore`) | 293,120 | — |
| 縮小全発動時 (= `calcMaxScore`) | **420,279** | +127,159 |
| 算術期待値 (= `calcExpectedScore.finalScore`) | **344,497** | +51,377 |
| MC 平均 (`runSimulation.mean`, SEED=42, iter=100) | **344,240** | +51,120 |
| MC 最小 (`mcMin`) | 317,808 | +24,688 |
| MC 最大 (`mcMax`) | 390,305 | +97,185 |
| MC 標準偏差 (`stddev`) | 13,840 | — |

### 縮小カバー率の計算ロジック (`calcShrinkCoverage`)

1. **発動回数の上限**: `floor(notesCount / skill.count) = floor(407(notes_20を除外) / 20(Lv5の場合)) = 20 回`
2. **スコアアップ対象の時間**: `20 回　※ 4秒 = 80秒`
3. **最大縮小カバー率**: `80秒 / MonstaerGenetation 104秒`
3. **縮小カバー率期待値**: `80秒*40% / MonstaerGenetation 104秒`
4. **最大縮小スコアアップ**: `notes_20を除外分を除外した基本スコア * 1.6(Lv5の場合) * カバー率（最大100%）`
5. **縮小スコアアップ期待値**: `notes_20を除外分を除外した基本スコア * 1.6(Lv5の場合) * カバー率（最大100%）* 40%`
