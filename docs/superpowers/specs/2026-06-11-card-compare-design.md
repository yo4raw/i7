# 衣装比較ページ（/card-compare/）設計

- 日付: 2026-06-11
- ステータス: ユーザー承認済み（モックアップ確認込み）
- 関連 ADR: [0007](../../adr/0007-card-compare-page.md)

## 背景と目的

衣装単体の「強さ」は属性値・スキル・曲に依存し、衣装が並んだときにどちらがスコアを伸ばすか判断しにくい。特に判定縮小スキルはデッキ全体（2枚以上で発動・並び順）に依存するため、単一の数値での比較がそもそも定義できない。

そこで前提条件を明示的に固定した上で、スキル種別ごとに適切な比較軸で全衣装を視覚的にランキング表示する専用ページを新設する。

## 決定事項（前提条件）

| 項目 | 決定 |
|------|------|
| 対象レアリティ | UR のみ |
| 表示範囲 | デフォルト「所持のみ」（`i7_card_counts` > 0）。全件への切替トグルあり。所持登録が空なら自動的に全件表示 |
| 楽曲 | セレクタで選択可能。初期選択は DIAMOND FUSION |
| 判定 | 全ノーツ Perfect 前提 |
| センタースキル | 完全に無視（非センター前提で比較） |
| 固有ブローチ | 常に装備込みで計算 |
| イベント特効 | デフォルトオフ。開催中イベントがある場合のみトグルを表示し、オンで特効ボーナスを属性値に反映 |
| 状態の永続化 | しない（毎回初期状態で表示。YAGNI） |

## スキル種別のグループ分け

比較の文法が異なるため、タブで2グループに分ける。

1. **スコアアップタブ**: スコアアップ（Perfect）/（コンボ）/（タイマー）に加え、判定補正系（MISS→Good / BAD以上をPerfectに変更）とスキルなしも含める。Perfect 前提では判定補正系・スキルなしのスキル期待値は 0 だが、属性値だけで強い衣装も比較対象にするため
2. **判定縮小タブ**: 判定縮小（Perfect）/（コンボ）/（タイマー）

縮小とスコアアップを横並びで比較する場面はない（役割が違う）ため、単一ランキングへの統合はしない。

## 計算モデル（1枚あたり・デッキ非依存）

### スコアアップ系: 期待スコア = 属性値由来スコア + スキル期待値

- **属性値由来スコア** = 選択曲の全ノーツについて Σ `calcNoteScore(そのカードの該当属性値, note)`。属性値は固有ブローチのステータス補正・特効ボーナス（オン時）適用後の値
- **スキル期待値** = `floor(floor(発動機会 ÷ count) × per/100 × value)`。発動機会はタイマー系なら曲秒数、それ以外はノーツ数。既存 `calcCardSkillExpected`（`src/lib/score/simulation.ts`）と同一の式
- `calcNoteScore` は属性値にほぼ線形（floor 2回の誤差のみ）なので、デッキを組まずに1枚分を正確に計算できる

### 判定縮小系: 多段ソート（単一数値化しない）

縮小はデッキ依存で期待スコアが定義できないため、実際の評価基準（秒数 → 必要コンボ数 → 確率 → 属性値）に合わせた優先順位の多段ソートで比較する:

1. 効果秒数（降順）
2. 選択曲での最大発動回数 = `floor(発動機会 ÷ count)`（降順）。コンボ系同士では「必要コンボ数の昇順」と同値であり、タイマー系（count = 秒数、発動機会 = 曲秒数）も同じ軸で比較できる
3. 発動確率（降順）
4. 属性値合計（降順）

同率（縦積み）の判定はキー 1〜3 が一致した場合とする。ラベルには生のスペック（必要コンボ数またはタイマー秒数・確率）を表示する。

## ページ構成

- ルート: `src/pages/card-compare/index.astro` + `src/components/CardCompare.svelte`
- ビルド時プリレンダリング + クライアント JS。カード・楽曲データは GViz からクライアントフェッチ（既存パターン踏襲）
- ナビゲーションメニューに「衣装比較」を追加（スコア計算の近く）
- 共通コントロール（タブの外側）: 楽曲セレクタ / 所持・全件トグル / 特効トグル / タブ切替

## UI

### スコアアップタブ

- 期待スコア合計の降順に並ぶ**積み上げ縦棒グラフ**。HTML/CSS（flexbox）で構築し、横スクロールで全件閲覧（SVG は使わない: サムネ埋め込み・タップ操作・レスポンシブ対応が容易なため）
- 各棒は2色積み上げ: 属性値由来スコア + スキル期待値。棒の上に合計値ラベル（「12.3万」形式の省略表記）
- 棒の下にカードサムネ（`loading="lazy"`）+ 属性色の枠線（赤 Shout / 緑 Beat / 青 Melody）+ スキル種別ラベル。特効オン時は特効バッジ表示

### 判定縮小タブ

- 棒の高さ = 効果秒数。多段ソート順に並べる
- 秒数・コンボ数（タイマー秒数）・確率がすべて同じ衣装は**1列に縦積み**して同率を視覚化
- 各列にコンボ数・確率・属性値合計のラベルを表示

### 詳細比較パネル

- 棒（サムネ）タップで選択トグル、**最大4枚**。選択中のみ画面下部に表示
- 表示内容: サムネ + 衣装名 / 属性値3種 / スキルスペック（種別・発動条件・効果値・確率） / 発動機会 / 期待スコア内訳（属性値由来・スキル期待値・合計）。最良値を強調表示
- スマホでは横スクロール対応

## 実装の分離とテスト

- 1枚あたりの計算ロジックは `src/lib/score/cardStrength.ts` として純粋関数で切り出し（UI 非依存）
  - 属性値由来スコア計算 / スコアアップ期待値計算 / 縮小ソートキー比較関数
- 単体テスト: `tests/unit/score/cardStrength.test.ts`（Vitest）
- E2E テスト: `tests/card-compare.test.ts`（ページ表示・タブ切替・グラフ描画・詳細パネル開閉）
- ダークテーマ: `dark:` バリアントをペアで指定（プロジェクト規約）

## エラー処理

- データフェッチ失敗時は既存ページと同様のエラーメッセージ表示
- 所持登録が空の場合は全件表示にフォールバックし、その旨を案内

## 検討した代替案

- **1ページ縦並び（タブなし）**: スコアアップと縮小のランキングを縦に並べる。UR 所持のみでも縦に長くなりすぎ、両グループを行き来する用途も薄いため不採用
- **スコア計算ページ内のモード追加**: 既存ページが編成 UI で既に複雑であり、比較は「編成前の検討」という別のジョブのため不採用
- **縮小の単一指標化**（発動率 × 秒数 × 縮小段階など）: 合成値の意味の説明が必要になり、実際の評価手順（秒数 → コンボ数 → 確率の辞書式比較）とも乖離するため不採用
- **センタースキルの表示モード追加**: 不要との判断で完全無視に確定
