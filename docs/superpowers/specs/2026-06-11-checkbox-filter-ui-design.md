# 衣装一覧フィルタのチップ+折りたたみ化 設計

- 日付: 2026-06-11
- 関連 ADR: [0005-checkbox-filter-ui](../../adr/0005-checkbox-filter-ui.md)

## 背景・目的

衣装一覧（`/cards/`）の絞り込み条件は `<select multiple>`（複数選択リストボックス）5 つで構成されている。複数選択に Ctrl/Cmd+クリックが必要でモバイルでは OS 依存の挙動になるなど選択しにくいため、タップで切り替えられるチップ（トグルボタン）ベースの UI に置き換える。

モックアップ 3 案（A: 全チップ常時表示 / B: ドロップダウン式チェックボックス / C: ハイブリッド）を比較し、案 C を採用した。

## UI 仕様

### 常時表示チップ（選択肢が少ないカテゴリ）

- 対象: レアリティ（5）・属性（3）・特効（3、開催中イベントがあるときのみ表示）
- `<button type="button" aria-pressed>` のトグルチップ。選択中は「✓」付き塗りつぶし表示
- 属性チップは選択時に属性色で塗る（Shout=赤 / Beat=緑 / Melody=青）。非選択時は他チップと同じ中立色
- その他のチップの選択色はサイトのアクセント（indigo）

### 折りたたみ（選択肢が多いカテゴリ）

- 対象: キャラクター（16）・スキルタイプ（11）
- ネイティブ `<details>/<summary>` で実装（外側クリック検知などの独自 JS を持たない）
- `<summary>` に選択件数バッジ（1 件以上のとき indigo の丸バッジ）を表示
- 初期状態: 閉。ただし URL パラメータ復元等でそのカテゴリに選択済み条件がある場合のみ初期展開する（マウント時に一度だけ判定し、以後の開閉はユーザー操作に委ねる）
- キャラクターは CHARACTER_GROUPS（IDOLiSH7 / TRIGGER / Re:vale / ŹOOĻ）のグループ行で並べ、行頭に点線枠のグループチップを置く
  - グループチップをタップすると所属メンバーを一括トグル: 全員選択中なら全員解除、それ以外は全員選択
  - グループチップ自体の表示状態は「所属メンバー全員が選択中」のとき選択表示

### 変更しないもの

- フィルタ状態（`raritySet` 等の `Set<string>`）、絞り込みロジック、URL パラメータ同期（`rarity` / `attr` / `char` / `skill` / `bonus`）
- 名前検索・ソート（`<select>` のまま）・条件リセット・件数表示・リスト/タイル切替
- ダークモード対応は既存規約どおり `dark:` バリアントをペアで指定

## 実装構成

- 新規コンポーネント `src/components/cards/CardFilterChips.svelte`
  - Props: ラベル・選択肢リスト・選択 Set（`$bindable`）。キャラクター用のグループ表示と一括選択はこのコンポーネント内で `groups` Prop により対応
  - 単純な「チップ列」と「details 折りたたみ + チップ列」を 1 コンポーネントで賄う（`collapsible` Prop）
- `CardList.svelte` は `<select multiple>` 5 つを `CardFilterChips` の呼び出しに置き換えるのみ。`<script>` 部のフィルタロジックは不変

## テスト

- `tests/card-list.test.ts` は旧ベースパス前提で `playwright.config.ts` の `testIgnore` により除外済みのため、本変更での改修はしない
- 単体テスト（`npm run test:unit`）はフィルタロジック非変更のため影響なし。実行して通過を確認する
- 動作確認は `npm run dev` + ブラウザ操作（チップ切替・グループ一括選択・URL 復元時の初期展開・ダークモード）

## 検討した代替案

- 案 A（全チップ常時表示）: 操作は最速だがスキルタイプ 11 種の長文チップで縦に長くなり一覧が押し出されるため不採用
- 案 B（ドロップダウン式チェックボックス）: 省スペースだが切替に 2 タップ必要で、外側クリック検知など実装も複雑なため不採用
