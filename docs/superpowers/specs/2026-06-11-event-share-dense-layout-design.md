# イベント SNS 共有パネルの高密度レイアウト設計

日付: 2026-06-11
ステータス: 承認済み

## 背景

イベント詳細配下の SNS 共有ページ(`events/[id]/share/ur|ssr`)は、現在 3〜5 列・`gap-2` のグリッドにカード名 2 行 + キャラ名のテキストを付けて表示しており、SNS で共有する画像としては疎すぎる。非公式 DB の特効まとめ画像(約 10 列・画像密着・スキル情報をセル表示)のような高密度レイアウトに変更したい。

ブラウザ上のモックアップ比較(視覚コンパニオン)でユーザーが以下を選択済み:

- レイアウト案 A「参考画像準拠」: 衣装名を画像の上、スキル種別 + 発動条件を画像の下に 2 段セルで表示
- 発動条件セルの背景は種別を問わず黄色で統一(コンボ/Perfect/タイマーの色分けはしない)
- 属性バッジ(Shout/Beat/Melody)は画像左上に重ねて維持
- 列数は 6 列(50 着級の銀特効でも読みやすさを優先)

## 変更対象

`src/components/EventSharePanel.astro`(UR / SSR 両共有ページが使用)。
ダウンロード機構(`EventShareImage.svelte` / modern-screenshot)と OG メタ・ページルーティングは変更しない。

## レイアウト仕様

- ヘッダー帯(インディゴ): イベント名 + レアリティ + イベント種別 + 開催期間 + サイト名 — 現行維持
- 金特効 / 銀特効セクション見出し(チップ + 着数 + 効果サマリー) — 現行維持
- グリッド: **常に 6 列・`gap-px`**。ブレークポイントで列数を変えない(ダウンロード画像の見た目を端末によらず一定にするため)
- パネル幅: `max-w-3xl` → `max-w-2xl`
- 各セル構成(上から):
  1. 衣装名(約 10px、中央寄せ、1 行 truncate)
  2. サムネイル画像 + 左上に属性バッジ(`ATTR_BADGE_BG` の色)
  3. スキル種別セル(太字・中央寄せ・2 行想定)
  4. 発動条件セル(黄色背景・太字): `ap_skill_req` の値
- 判定縮小はスキル種別セルを**ピンク背景 + 白文字**で強調
- 削除: 画像下のカード名 2 行表示、キャラ名行、レアリティバッジ(ページが UR/SSR 別のため冗長)

## スキル表示の導出

`src/lib/score/skillFormatter.ts` に `formatSkillBadge(skillType)` を追加する。
正規化済み `ap_skill_type` から短縮ラベルと縮小フラグを返す:

| ap_skill_type | ラベル | isShrink |
|---|---|---|
| スコアアップ（…） | スコアアップ | false |
| 判定縮小（…） | 判定縮小 | true |
| BAD以上をPerfectに変更 | BAD→Perfect | false |
| MISS以上をPerfectに変更 | MISS→Perfect | false |
| MISS→Perfect | MISS→Perfect | false |
| MISS→Good | MISS→Good | false |
| 判定拡大スコアダウン | 判定拡大 | false |
| null / 未知 | -(未知はそのまま表示) | false |

発動条件は `ap_skill_req` をそのまま表示する。

## スタイリング

Tailwind ユーティリティで実装し、ダークバリアントをペアで指定する。
発動条件セルは両モード共通の黄色系(`bg-yellow-300 text-yellow-950` 程度)、判定縮小は `bg-pink-500 text-white`。

## テスト

- `tests/unit/score/skillFormatter.test.ts` で `formatSkillBadge` の全種別をカバー(Vitest)
- `npm run dev` + Playwright MCP で `events/236/share/ur/`(開催中・銀特効多数)のライト/ダーク表示を確認

## 検討した代替案

- 案 B「オーバーレイ型」(画像下部に半透明バー、最高密度) — 画像が隠れるため不採用
- 案 C「1 行コンパクト」(スキル/条件を 1 行凝縮) — 視認性で案 A に劣るため不採用
- 列数 8 / 10 — 文字が拡大前提になるため不採用
- 発動条件の色分け(タイマー水色等) — ユーザー判断で黄色統一
